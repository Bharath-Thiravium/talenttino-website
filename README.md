# Talentteno Institute

Clean project layout with separate frontend, backend, and database folders.

## Folder Structure

```text
talentteno/
├── frontend/              Public website pages, assets, uploads
├── backend/               PHP admin panel, DB config, optional Node API
│   ├── admin/             Admin panel pages
│   ├── includes/          Shared PHP backend config
│   └── node/              Node.js + MySQL API
├── database/              MySQL schema and seed data
│   └── database_setup.sql
└── README.md
```

## Run PHP Website

```bash
php -S 127.0.0.1:8000
```

For media uploads up to 100 MB, start the local PHP server with matching upload limits:

```bash
php -d upload_max_filesize=100M -d post_max_size=110M -S 127.0.0.1:8000
```

Open:

- Frontend: `http://127.0.0.1:8000/frontend/index.php`
- Admin: `http://127.0.0.1:8000/backend/admin/login.php`

Default admin:

- Username: `admin`
- Password: `password`

## Database

Create and import for local development:

```bash
chmod +x database/setup_local_mysql.sh
database/setup_local_mysql.sh
```

On XAMPP, start MySQL from XAMPP first. The setup script will use `/opt/lampp/bin/mysql` automatically when it is available.

DB settings are in:

```text
backend/includes/db.php
```

You can also use environment variables:

```text
DB_HOST
DB_PORT
DB_USER
DB_PASS or DB_PASSWORD
DB_NAME
DB_SOCKET
```

Default local development credentials:

```text
DB_USER=talentteno_app
DB_PASS=talentteno_local_password
DB_NAME=talentteno_db
```

## Run Node Backend

```bash
cd backend/node
npm install
npm run start
```

Node API:

```text
http://127.0.0.1:5000/api/health
```

## Notes

- `database/runtime/` is local MySQL generated data and should not be committed.
- `backend/node/node_modules/` is generated by `npm install`.
- Public brochure uploads belong in `frontend/uploads/brochures/`.
